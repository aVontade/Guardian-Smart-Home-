# Guardian Smart Home - Development Guide for AI Assistants\n\n## Project Overview\n\n**Guardian Smart Home** is a high-end smart home security system dashboard built with React 19, Vite, Tailwind CSS, and Google Gemini AI integration. This guide helps AI assistants (Claude, Copilot, etc.) understand the project structure and contribute effectively.\n\n## Project Context\n\n- **Repository**: aVontade/Guardian-Smart-Home-\n- **Owner**: Hamede Ali (@aVontade)\n- **Language**: TypeScript + React\n- **Build Tool**: Vite 6\n- **Styling**: Tailwind CSS 4\n- **Backend**: Express.js + Google Gemini AI\n- **License**: Apache 2.0\n\n## Core Architecture\n\n### Frontend Stack\n- **React 19**: Main UI framework with hooks\n- **Vite 6**: Development server and build tool\n- **Tailwind CSS 4**: Utility-first styling\n- **Motion**: Animation library for smooth UX\n- **TypeScript**: Full type safety\n\n### Backend Stack\n- **Express.js**: REST API server\n- **Google Gemini AI**: AI insights generation\n- **Vite Middleware**: Dev server integration\n\n## Key Files & Directories\n\n```\nsrc/\n├── App.tsx                 # Main dashboard (entry point)\n├── types.ts               # TypeScript interfaces\n├── components/            # React components\n│   ├── Header.tsx\n│   ├── SecurityStatusRing.tsx\n│   ├── DeviceGrid.tsx\n│   ├── CameraSection.tsx\n│   ├── InteractiveFloorPlan.tsx\n│   ├── AIInsights.tsx\n│   ├── Timeline.tsx\n│   ├── EmergencyPanel.tsx\n│   └── NetworkScanner.tsx\n├── main.tsx              # React entry point\n├── index.html            # HTML entry point\n└── index.css             # Global styles\n\nserver.ts                # Express server with Gemini integration\nvite.config.ts          # Vite configuration\ntsconfig.json           # TypeScript configuration\npackage.json            # Dependencies\n```\n\n## Data Models\n\n### SecurityStatus\n```typescript\ntype SecurityStatus = 'armed' | 'disarmed' | 'alert' | 'warning';\n```\n\n### Device\n```typescript\ninterface Device {\n  id: string;\n  name: string;\n  type: 'camera' | 'lock' | 'sensor' | 'light' | 'thermostat' | 'plug' | 'speaker' | 'vacuum' | 'fridge';\n  status: 'active' | 'inactive' | 'locked' | 'unlocked' | 'open' | 'closed' | 'online' | 'offline';\n  value?: string | number;\n  battery?: number;\n  location: string;\n  diagnostics?: {\n    signalStrength: number;\n    uptime: string;\n    firmware: string;\n    ipAddress: string;\n  };\n}\n```\n\n### SecurityEvent\n```typescript\ninterface SecurityEvent {\n  id: string;\n  timestamp: string;\n  type: string;\n  message: string;\n  location: string;\n  severity: 'info' | 'warning' | 'critical';\n  thumbnail?: string;\n}\n```\n\n### UserProfile\n```typescript\ninterface UserProfile {\n  id: string;\n  name: string;\n  role: 'admin' | 'family' | 'guest';\n  avatar: string;\n}\n```\n\n## Component Structure\n\nAll components follow this pattern:\n\n```typescript\nimport { ComponentType } from 'react';\nimport { motion } from 'motion/react';\n\ninterface Props {\n  // Type-safe props\n}\n\nexport function ComponentName(props: Props) {\n  return (\n    // JSX with Tailwind classes\n  );\n}\n```\n\n### Key Components\n\n1. **Header** - Navigation and user profile\n2. **SecurityStatusRing** - Visual security status indicator\n3. **DeviceGrid** - Device control interface\n4. dev\n```\nStarts Vite dev server at http://localhost:3000\n\n### Build\n```bash\nnpm run build\n```\nCreates optimized production build\n\n### Production\n```bash\nnpm start\n```\nRuns compiled server\n\n### Type Check\n```bash\nnpm run lint\n```\nValidates TypeScript\n\n## Styling Guidelines\n\nWe use **Tailwind CSS 4** with these conventions:\n\n- **Colors**: Use semantic color names (blue, cyan, emerald, red)\n- **Spacing**: Consistent with 8px base unit\n- **Rounded**: Use rounded-full, rounded-3xl, rounded-xl, rounded-lg\n- **Backdrop**: Use backdrop-blur-xl for frosted glass effects\n- **Borders**: Use border-white/5 for subtle dividers\n- **Dark Mode**: Base background is #09090b, text is white\n- **Opacity**: Use /5, /10, /20, /30, /40 for fine control\n\n### Example:\n```tsx\n<div className=\"bg-white/5 border border-white/5 rounded-3xl p-8 backdrop-blur-xl\">\n  <h2 className=\"text-3xl font-light tracking-tight mb-2\">Title</h2>\n  <p className=\"text-white/30 text-sm leading-relaxed\">Description</p>\n</div>\n```\n\n## Animation Patterns\n\nUsing **Motion** library:\n\n```tsx\n// Pulsing indicator\n<motion.span\n  animate={{ opacity: [0.4, 1, 0.4] }}\n  transition={{ duration: 2, repeat: Infinity }}\n  className=\"w-1.5 h-1.5 rounded-full bg-emerald-400\"\n/>\n\n// Smooth transition\n<motion.div\n  initial={{ opacity: 0, y: 10 }}\n  animate={{ opacity: 1, y: 0 }}\n  transition={{ duration: 0.3 }}\n>\n  Content\n</motion.div>\n```\n\n## API Integration\n\n### Health Endpoint\n```typescript\nGET /api/health\nResponse: { status: \"ok\" }\n```\n\n### AI Insights Endpoint\n```typescript\nPOST /api/insights\nBody: {\n  status: SecurityStatus;\n  events: SecurityEvent[];\n}\nResponse: {\n  insight: string;\n  suggestion: string;\n}\n```\n\n## Code Style & Best Practices\n\n### TypeScript\n- Always use interfaces over types (unless discriminated unions)\n- Export named exports, not defaults\n- Use strict null checks\n- Avoid any types\n\n### React\n- Functional components only\n- Use hooks (useState, useEffect, useContext)\n- Prop drilling: maximum 2 levels deep\n- Memoize expensive computations\n\n### Naming Conventions\n- Components: PascalCase (e.g., SecurityStatusRing)\n- Functions: camelCase (e.g., handleToggleDevice)\n- Constants: UPPER_SNAKE_CASE (e.g., MAX_DEVICES)\n- Types/Interfaces: PascalCase (e.g., Device, SecurityEvent)\n\n### File Organization\n- One component per file\n- Components in `src/components/`\n- Types in `src/types.ts`\n- Utilities in `src/utils/` (if needed)\n\n## Common Tasks\n\n### Adding a New Device Type\n1. Update `Device['type']` union in `types.ts`\n2. Add icon in `DeviceGrid.tsx`\n3. Add status logic in `App.tsx` handleToggleDevice\n4. Update type checking in relevant components\n\n### Adding a New Component\n1. Create file in `src/components/ComponentName.tsx`\n2. Define props interface\n3. Export named function component\n4. Import and use in `App.tsx`\n\n### Styling a New Component\n1. Use Tailwind classes exclusively\n2. Follow color scheme: blues, cyans, whites with opacity\n3. Use backdrop-blur-xl for glass effects\n4. Add border-white/5 for subtle divisions\n5. Use rounded-3xl for card corners\n\n### Connecting to AI Insights\n1. Fetch `/api/insights` with POST\n2. Send current security status and events\n3. Parse JSON response\n4. Display insight and suggestion in UI\n\n## Environment Variables\n\n```env\nGEMINI_API_KEY=your_api_key_here\nNODE_ENV=development|production\n```\n\n## Performance Considerations\n\n- Device grid updates use state mapping, not re-renders\n- Animations use GPU acceleration (transform, opacity)\n- Images use lazy loading where applicable\n- Component memoization for expensive computations\n\n## Security Considerations\n\n- All sensitive data handled server-side\n- AES-256 encryption for data at rest\n- Environment variables for API keys\n- TypeScript for compile-time safety\n- Role-based access control in types\n\n## Testing (Future)\n\nWhen tests are added:\n- Use Vitest for unit tests\n- React Testing Library for component tests\n- E2E tests with Playwright\n\n## Deployment\n\n### Replacement Issues\n- Set `DISABLE_HMR=false` in environment\n- Restart dev server\n\n### TypeScript Errors\n- Run `npm run lint` to check all files\n- Verify imports have types\n- Check strict null
